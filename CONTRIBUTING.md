# Contributing to Maayar Documentation

This repository accepts **documentation, architecture proposals, and design feedback** only.

---

## Welcome

| Contribution | Examples |
|--------------|----------|
| Documentation improvements | Clarity, accuracy, completeness |
| Architecture proposals | ADRs, design alternatives |
| Diagram enhancements | ASCII diagram corrections |
| Workflow documentation | Pipeline clarifications |

## Not Accepted

- Production source code
- Proprietary algorithms or classification parameters
- Model configurations or prompt artifacts
- Secrets or environment-specific values

---

## Process

1. Open an issue for significant changes
2. Fork and branch: `docs/topic` or `adr/number-title`
3. Follow enterprise terminology (see [Glossary](./docs/reference/glossary.md))
4. Submit PR with clear description

## ADRs

Copy [0000-template.md](./architecture/decisions/0000-template.md), number sequentially, submit to `architecture/decisions/`.

## Standards

- Use ASCII diagrams for architecture flows
- Cross-link related documentation
- Do not include proprietary implementation details
- Every folder must have a README.md

→ [Code of Conduct](./CODE_OF_CONDUCT.md)
